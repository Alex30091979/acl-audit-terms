- **Даты коммитов** (author date, commit date) — неизменяемая хронология изменений.

**Где смотреть:**


## 3. Подтверждённая идентичность (Verified)

### GitHub
- Привяжите к аккаунту **подтверждённый email** (в т.ч. `...@users.noreply.github.com`).
- Коммиты с этого email помечаются как **Verified** на GitHub.
- Настройка: GitHub → Settings → Emails → Add / Verify.

### GitLab
- Аналогично: **подтверждённый email** в GitLab.
- Коммиты от этого email считаются связанными с вашим аккаунтом.

Использование **одного и того же** подтверждённого email в Git, GitHub и GitLab усиливает доказательства.

## 4. Подписанные коммиты (GPG) — настроено

GPG-подпись связывает коммит с вашим ключом и существенно усиливает доказательства авторства.

**Уже сделано:**
- Создан GPG-ключ Ed25519 (Key ID: `2058C154F18EA518`).
- Git настроен: `user.signingkey`, `commit.gpgsign true`, `gpg.program`.
- Подписан тестовый коммит, подпись проверена.

**Вам осталось:** добавить открытый ключ в [GitHub](https://github.com/settings/keys) и [GitLab](https://gitlab.com/-/user_settings/gpg_keys). Открытый ключ лежит в файле `gpg-public-key.asc` в корне проекта. Подробности в [GPG_SETUP_DONE.md](GPG_SETUP_DONE.md).

**Важно (Windows):** подписанные коммиты нужно делать из **Git Bash** (не из PowerShell), иначе gpg-agent не запускается. См. `GPG_SETUP_DONE.md`.

После добавления ключа в GitHub/GitLab подписанные коммиты отображаются как **Verified**.

## 5. Дублирование репозиториев (GitHub + GitLab)

- Один и тот же код и история хранятся в **двух** независимых площадках.
- Оба репозитория привязаны к вашим аккаунтам (Alex30091979).
- При споре можно ссылаться на оба как на независимые свидетельства наличия кода и авторства у вас на определённую дату.

Регулярно синхронизируйте ветки:
```bash
git push origin master
git push gitlab master
```

## 6. Релизы и теги

- Создавайте **теги** (например, `v6.5.12`) и **релизы** в GitHub и GitLab.
- У каждого релиза есть дата и привязка к коммиту.
- Это фиксирует «снимок» кода на конкретную дату и усиливает хронологию.
